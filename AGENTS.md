# AI Team Rules — weekly-meal-planner

## Lead Rule
**Claude Code is the Lead Developer and Architect.**
No large change is implemented without a task brief and QA review.
All changes go directly to `main` branch and are pushed immediately after build verification.

## Project Context
- Static site with 204 recipes in 14 languages (ro, en, es, fr, de, pt, ru, ar, zh, ja, hi, tr, it, ko)
- ~3670 HTML pages generated by `scripts/generate-content.mjs` from `public/js/recipes.js` (recipes + plans + 644 cuisine hubs + indexes)
- Stripe payments + Supabase auth/subscriptions
- Serverless API on Vercel (`/api/*.js`)
- Deployed: Vercel + GitHub Pages (CNAME configured)
- Build command: `npm run build` (JS + CSS + content generation)
- Content only: `npm run content`

## Critical Rules (apply to ALL agents)
1. Never modify `public/js/recipes.js` without running `npm run content` and verifying ~3670 pages
2. Never commit if `python3` curly-quote check finds U+2018/U+2019 in recipes.js
3. Never skip `npm run build` before committing JS/CSS changes
4. Never create separate branches — commit directly to `main` and push
5. Never touch `.env` or `.env.local` — these are gitignored secrets
6. Recipes with 9+ ingredients need explicit `servings: 4` override
7. Every multilingual field must include `hi` (Hindi) — older recipes are missing it

## How to Use Claude Code (Lead)

```
Read AGENTS.md and docs/ai/*.md.
You are the Lead Developer and Architect.
Do not start coding until you create a task plan.
For each task:
1. Identify files affected
2. Implement the change
3. Run build checks (npm run content OR npm run build)
4. Verify output (page count, no curly quotes, no JS errors)
5. Commit with descriptive message
6. Push to main
```

### ⚠️ Non-blocking issues (pre-existing, outside generator scope)

#### Issue 1 — `© 2025` in 15 static homepage files
**Files:** `public/ro/index.html`, `public/en/index.html`, `public/index.html`, and 12 other `public/{lang}/index.html` files.

These are static app-shell HTML files, not generated by `generate-content.mjs`. The generator's `makeFooter()` fix (Phase 1) only applies to generated pages. These files were previously updated for hero pill copy but footer year was not addressed at that time.

**Impact:** Cosmetic only. Homepages show `© 2025` in footer. No SEO or functional impact.
**Fix:** Update footer year in each of the 15 static files. Safe, trivial change.

#### Issue 2 — `"1 PDF per day"` in `public/pricing/index.html`
**File:** `public/pricing/index.html` — a root-level static pricing page (distinct from the generated `/en/pricing/index.html`).

The string appears in a JSON-LD `"description"` field:
```
"description": "7-day meal planner, auto shopping list, 175 recipes, 1 PDF per day."
```
This root `/pricing/` page is a legacy static file that predates the generated pricing pages. It is not linked from any current nav (Premium nav links to `/en/pricing/` and `/ro/premium/`, not `/pricing/`). Whether it remains indexed is unknown.

**Impact:** Low. Page may be directly bookmarked or indexed but is not in the main nav flow.
**Fix:** Update the JSON-LD description string in `public/pricing/index.html`.

#### Issue 3 — Garbled slugs from non-ASCII recipe EN names
Two current recipe pages have garbled-looking (but functionally valid) slugs:
- `public/ro/retete/sm-rrebr-d/` — "Smørrebrød" (Danish open sandwich)
- `public/ro/retete/bacalhau-br-s/` — "Bacalhau à Brás" (Portuguese dish)

These are **current**, generator-produced slugs (not stale). The slug function strips `ø`, `à` etc. to `-`. The pages exist, are in the sitemap, and function correctly. They are linked from plan pages (Smørrebrød appears in the world-tour plan).

**Impact:** User-facing URLs are slightly ugly; minor SEO signal loss from non-descriptive hyphens. No broken links.
**Fix:** Change `name.en` for each recipe to an ASCII transliteration (e.g. `"Smorrebrod"`, `"Bacalhau a Bras"`). Requires the same Phase E recipe audit process.

#### Issue 4 — GitHub Actions CI status unknown
`gh` CLI not authenticated in this environment. All changes were pushed to `origin/main`. The changes are static HTML + one script edit — no dependencies changed, no npm packages modified. Risk of CI failure is very low.

#### Issue 5 — `hi:` translation missing in id:119 (Kottbullar stub)
The Kottbullar recipe (id:119, now at `/hi/recipes/kottbullar/`) has `name.hi` absent in `recipes.js`. The generator falls back to the EN name `"Kottbullar"`. Page functions correctly but shows a non-localized name in Hindi. Pre-existing data quality issue.

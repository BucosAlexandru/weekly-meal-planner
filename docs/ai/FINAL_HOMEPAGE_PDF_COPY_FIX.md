# Final Homepage PDF Copy Fix

_Implemented: 2026-05-18_
_Commit: `8f18f971f`_
_Owner: Claude Code (Lead Developer)_

---

## Problem

Every localized app homepage (`/ro/`, `/en/`, `/fr/`, etc.) and the root `/` showed a hero pill badge with misleading text:

| File | Old text |
|------|---------|
| `public/ro/index.html` | `1 PDF gratuit/zi` |
| All other 14 homepage files | `1 free PDF/day` |

The JSON-LD `"description"` fields (app and Free Plan offer) in 12 of these files made the same claim.

**Why this was wrong:**
1. **"Full"/"1"** — the free PDF covers exactly **2 of 7 days** (`const freeDays = 2` in `app.js:484`). It is a preview, not a complete plan.
2. **"Per day"** — there is no daily download cap. Free users can download as many times as they like; the content is always limited to the first 2 days regardless.

**Why previous fixes missed this:**
The Pricing Consistency Fix (commit `105831bc7`) targeted `renderPricingSection()` in `app.js` (the collapsible in-page pricing cards) and `PRICING_COPY` in `generate-content.mjs` (standalone pricing pages). The hero pills and JSON-LD are **hardcoded static HTML in each localized homepage file** — not generated by any script, invisible to `npm run content`.

---

## Scope

Static HTML copy changes only — 15 files, no scripts, no `app.js`, no `i18n.js`, no API, no recipes, no payment.

# Recipe Image Backlog

109 recipes in `public/js/recipes.js` have no entry in `public/js/recipe-images.js`. Pages for these IDs currently fall back to the site-wide cover image (`/images/cover2.jpg`). Until each is mapped, every untouched recipe page and every social-share preview for those URLs shows the same generic cover — the strongest visual cue that the catalogue is templated rather than handcrafted.

Source of truth: `public/js/recipe-images.js` is auto-generated by the external image-fetch tooling (Spoonacular API + Wikipedia commons). Do not hand-edit. Re-run the tooling with the list below to populate them.

## Process

1. Run the image-fetch tool against each ID. Prefer Spoonacular (consistent 312×231 crops); fall back to Wikipedia commons thumbnails only for dishes Spoonacular doesn't index (regional/obscure).
2. Verify each returned URL renders before merging — Spoonacular has been known to return 404 for recently removed recipes.
3. After regenerating `recipe-images.js`, run `npm run content && npm run build` and visually QA at least one of the previously-missing pages per language family (Latin, Cyrillic, Arabic, CJK).
